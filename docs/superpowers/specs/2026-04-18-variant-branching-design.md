# バリアント分岐機能 設計書

## 概要

Object / Pane / Screen エディタ上で、モデルの複数案（バリアント）を分岐させて試行錯誤し、比較・採用できる機能。Figmaで「まるごと横に複製して別案を作る」ワークフローを、このエディタに適した形で実現する。

## ワークフロー

1. エディタで編集中に「ここから別方向も試したい」と思ったら **Duplicate** で現在のモデルをフォーク
2. 各バリアント（Option A, Option B, ...）を切り替えながら独立して編集
3. **Split** で2つのバリアントを横並びで比較
4. **Keep** で1つを採用、他のバリアントは削除される
5. Objectタブで決定してからPaneタブへ、Paneで決定してからScreenへと進む

## データモデル

### 通常状態（分岐なし）

```json
{
  "objects": [ ... ],
  "views": [ ... ],
  "paneGraph": [ ... ],
  "screens": { ... }
}
```

### 分岐状態

```json
{
  "_variants": [
    {
      "id": "a",
      "name": "Option A",
      "active": true,
      "objects": [ ... ],
      "views": [ ... ],
      "paneGraph": [ ... ],
      "screens": { ... }
    },
    {
      "id": "b",
      "name": "Option B",
      "objects": [ ... ],
      "views": [ ... ],
      "paneGraph": [ ... ],
      "screens": { ... }
    }
  ]
}
```

### ルール

- 全バリアントが `_variants` 配列内に対等に格納される
- `active: true` のバリアントがエディタ上で現在選択中のもの
- 分岐時: トップレベルの `objects` / `views` / `paneGraph` / `screens` を `_variants[0]` に移動 → コピーして `_variants[1]` を追加 → トップレベルのキーを削除
- 採用時: 選んだバリアントのデータをトップレベルに展開 → `_variants` を削除
- バリアントが1つだけになったら `_variants` を解除して通常状態に戻る

## UI: サブタブバー

### レイアウト

```
[Object]  [Pane]  [Screen]                       ← メインタブ
[Option A] [Option B] [Option C] [+ Duplicate]  [Split]  [Keep]
```

### 表示条件

- 分岐がない状態ではサブタブバーは非表示
- 初回 Duplicate 時にサブタブバーが出現
- Keep または削除でバリアントが1つになったらサブタブバーが消える

### 操作一覧

| 操作 | トリガー | 動作 |
|------|---------|------|
| **切り替え** | サブタブをクリック | `active` を切り替え、キャンバスを再描画 |
| **Duplicate** | 「+ Duplicate」ボタン / 右クリックメニュー | 選択中のバリアントをコピーして新バリアント追加 |
| **リネーム** | サブタブをダブルクリック | インライン編集で名前変更 |
| **削除** | サブタブの「×」ボタン / 右クリックメニュー | 特定のバリアントを削除。最後の1つになったら `_variants` 解除 |
| **Split** | 「Split」ボタン | バリアント一覧から2つ選択 → キャンバスを左右分割 |
| **Keep** | 「Keep」ボタン | 確認ダイアログ → アクティブバリアントをトップレベルに展開、他を全削除 |

### 自動命名

- Duplicate時の自動命名: "Option A", "Option B", "Option C", ... とアルファベット順
- ダブルクリックで任意の名前にリネーム可能

## Split View

### 基本動作

1. 「Split」ボタンを押す → バリアント一覧から2つ選ぶ
2. 選んだ2つでキャンバスが左右分割
3. Split中、左右それぞれのバリアント名をクリックすると別のバリアントに差し替え可能
4. もう一度「Split」を押すと閉じる（トグル）

### Split中の編集

- 左右どちらもクリック・ドラッグで編集可能
- フォーカスしている側の枠線がハイライトされる（どちらを操作中か明示）
- Undo/Redo は統一履歴に従う（後述）

### Split中の操作

- 「Duplicate」使用可（フォーカス中のバリアントを複製）
- 「Keep」使用可（フォーカス中のバリアントを採用。Split解除→単一表示に戻る）
- パネル内のバリアント差し替え可（例: A vs B → A vs C）

## Undo/Redo

### 基本方針

履歴はタブ単位（Object / Pane / Screen）で統一管理。全バリアントの操作が1つの履歴スタックに入る。

### 履歴エントリの構造

各エントリに「どのバリアントでの操作か」を記録する。

### 動作ルール

1. Undoは常に直前の操作を戻す
2. 操作対象バリアントが現在の表示と異なる場合、**そのバリアントに画面を切り替えてから**戻す
3. Split中に表示外のバリアントのUndoが発生した場合、**Splitを閉じてからそのバリアントに切り替えて**戻す
4. Redoも同様のルールに従う
5. Duplicate操作自体も履歴に入る（UndoでDuplicate取り消し可能）

### Duplicate時の履歴

- Duplicate時、新バリアントに元の履歴はコピーしない
- 新バリアントはフォーク時点が履歴の起点となる

## 自動保存

- バリアント存在時は `_variants` 配列を含むJSONをそのまま保存
- 切り替え・編集・削除・Duplicate すべてが自動保存の対象
- 既存の500ms遅延の仕組みはそのまま維持

## 既存スキルとの互換性

| スキル | 動作 |
|--------|------|
| **edit** | そのまま動作。`_variants` がある状態でエディタを開き、バリアントUIが復元される |
| **generate** | `_variants` が存在する場合、「バリアントが存在しますが、上書きされます。よろしいですか？」と確認ダイアログを出す。OKなら `_variants` を削除して通常状態に上書き |

## ファイル読み込み

- JSONを読んだとき、トップレベルに `_variants` キーがあれば分岐状態として復元
- なければ通常状態（サブタブバー非表示）
- `_variants` の構造が壊れていた場合（手動編集ミス等）はエラー表示して読み込みを中断
