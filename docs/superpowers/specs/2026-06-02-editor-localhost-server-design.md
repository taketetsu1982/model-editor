# editor localhostローカルサーバ方式 設計

作成日: 2026-06-02
ブランチ: `feat/editor-localhost-server`

## 背景・目的

現在の `/edit` は `editor.html` を `file://` でブラウザに開き、ユーザーは**起動のたびに** ドラッグ&ドロップ または「Connect」ボタンから `product-model.json` を選び直す必要がある（接続後の自動保存自体は既に実装済み）。

この「毎回のファイル選択」を**完全になくす**ことが目的。

## 方針

editor.html を `file://` ではなく **localhost のローカルサーバ経由**で配信する。editor は決まったエンドポイントから対象JSONを `fetch` で自動読込し、保存はサーバ経由でファイルに直接書き戻す。これによりファイル選択ダイアログも権限ダイアログも一切不要になる。

`file://` で直接開いた場合の従来動作（ピッカー / D&D / File System Access API）は**フォールバックとして必ず残す**。サーバはあくまで上位の便利レイヤーとして追加する。

## 全体構成

```
/edit [path]   ← path 省略時は既定パスを自動検出
   │
   ├─ 1. Nodeで小さなローカルサーバを起動（バックグラウンド、追加依存なし）
   │      ・対象JSONの絶対パスを起動引数で受け取る
   │      ・GET  /          → editor.html を配信
   │      ・GET  /lib/*     → lib配下を配信
   │      ・GET  /model     → 対象JSONを返す（fetch自動読込用）
   │      ・PUT  /model     → 受け取ったJSONを対象パスに上書き保存
   │
   ├─ 2. open http://localhost:PORT/ でブラウザを開く
   │
   ├─ 3. editor は onload で fetch('/model') → 自動読込（選択ダイアログなし）
   │      保存は PUT /model → サーバがファイルに直書き（選択も権限も不要）
   │
   └─ 4. 編集完了 → サーバ停止 → Claude が対象JSONを読み戻す
```

## コンポーネント（責務分離）

| コンポーネント | 責務 | 依存 |
|---|---|---|
| `editors/server.js`（新規） | 静的配信 + `/model` の GET/PUT。対象パスは起動引数。**指定パス以外への書込み禁止**（パストラバーサル防止）。CORSは `localhost` のみ許可 | Node標準のみ（http, fs, path） |
| `editor.html` 読込ロジック | サーバ配信時は `fetch('/model')` で自動読込。`file://` 時は従来のピッカー/D&Dにフォールバック | 既存 `lib/file-io.js` |
| `editor.html` 保存ロジック | サーバ接続時は `PUT /model` で保存。`file://` 時は従来の File System Access API | 既存 `lib/file-io.js` |
| `edit` スキル | サーバ起動 → ブラウザ起動 → 完了後にサーバ停止＆JSON読み戻し | `server.js` |

### 動作モードの判定

editor.html はロード時に自身の配信元を判定する:

- `location.protocol === 'http:'`（= localhostサーバ経由）→ **サーバモード**: `fetch('/model')` で自動読込、`PUT /model` で保存
- `location.protocol === 'file:'` → **ファイルモード**: 従来通りピッカー / D&D / File System Access API

この判定により単一の `editor.html` が両モードで動作し、後方互換を保つ。

## データフロー

### 読込（自動）
1. ブラウザが `http://localhost:PORT/` を開く
2. editor.html の onload で `fetch('/model')` を実行
3. サーバが対象JSONファイルを読んでレスポンス
4. editor が既存の `loadJson(text, fileName)` に渡して描画

### 保存
1. ユーザーが編集 → 既存の `markModified()` → 自動保存スケジュール（既存ロジックを流用）
2. サーバモードでは保存処理が `PUT /model`（body = JSON文字列）を実行
3. サーバが**起動引数で指定された絶対パスにのみ**書き込む
4. 書込み成功で editor のステータスを `connected` 表示に更新

### 読み戻し（Claude側）
- 保存はファイルに直書きされるため、Claude は完了を待たずいつでも対象JSONを読める
- 「完了」指示でサーバを停止し、最終状態を読み戻して変更を確認

## 設計上の決定（デフォルト）

1. **ポート**: 固定 `8765`。使用中なら +1 して空きポートを探索する
2. **サーバ寿命**: `/edit` でバックグラウンド起動 → ユーザーが「完了」と言った時点で Claude が停止する
3. **パス省略時**: リポジトリ内の `product-model.json` を1件だけ自動検出して使用。0件または複数件ならパス指定を促す
4. **後方互換**: `file://` で直接開いても従来通り動作する

## エラーハンドリング

- **対象JSONが存在しない**: サーバ起動時に検出し、エラー終了して理由を表示（パス指定を促す）
- **`/model` GET失敗**: editor はサーバモードでも読込失敗時はファイルモードのフォールバックUI（Connectボタン）を表示
- **`PUT /model` 失敗**: editor にトーストでエラー表示。ファイル内容は壊さない（書込みは一時ファイル経由 → リネームの原子的書込みを検討）
- **ポート探索失敗**（連続使用中）: 一定回数で諦めてエラー表示
- **パストラバーサル / 範囲外書込み**: サーバは起動引数パス以外への書込みを拒否

## セキュリティ

- サーバは `127.0.0.1` のみにバインド（LAN公開しない。共有が必要になったら別途検討）
- 書込み先は起動引数の絶対パスに限定
- CORSは `localhost` 由来のみ許可

## テスト方針

- `editors/server.js`: GET `/model` が対象ファイルを返すこと、PUT `/model` が対象パスにのみ書くこと、範囲外パスを拒否することのユニットテスト（既存の `*.test.js` と同じ枠組み）
- `lib/file-io.js`: モード判定（http / file）に応じた読込・保存分岐のテスト
- 手動E2E: `/edit` 実行 → ブラウザで自動読込される → 編集して保存 → ファイルが更新される → file:// で開いた場合も従来通り動く

## 非目標（YAGNI）

- LAN内の他端末からのアクセス共有（今回は対象外）
- 複数JSONの同時編集
- サーバの常駐・デーモン化
